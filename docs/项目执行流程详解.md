## 一、执行入口总览

### 1.1 两个主要入口

```
项目根目录/
├── main.py           # 本地一键启动脚本（训练 + 启动）
└── app.py            # Streamlit 前端入口（用户界面）
```

**执行选择：**

| 入口 | 使用场景 | 执行内容 |
|------|----------|----------|
| `python main.py` | 本地开发/测试 | 检查数据 → 训练模型 → 启动前端 |
| `streamlit run app.py` | 生产环境 | 直接启动前端（模型需预先训练好） |

### 2.2 关键函数调用链

```python
# main.py 中的调用链

main()
└── check_all_layers()
    ├── check_data_layer()
    │   └── pd.read_csv('data/data.csv')
    │
    ├── check_model_layer()
    │   └── 检查 models/*.bin 文件
    │
```

```python

```

## 三、app.py 执行流程

### 3.1 Streamlit 应用结构

```python
# app.py 执行流程

   └── from src.services import ForecastService, TrainingService

2. 页面配置
   └── st.set_page_config(...)

   ├── get_service()      # 创建 ForecastService 单例
```

## 五、关键类与接口

### 5.1 核心类继承关系

```
# 模型层继承关系

BaseForecastModel (抽象基类)
├── LSTMForecastModel
├── GRUForecastModel
├── XGBoostForecastModel
└── MoiraiForecastModel


# 服务层组合关系

ForecastService
└── ForecasterManager
    ├── ModelService
    └── HorizonTradeAdvisor
```

### 5.2 关键接口定义

```python
# 模型接口（src/models/base_model.py）
class BaseForecastModel:
    def train(self, X_train, y_train, **kwargs) -> Dict[str, Any]
    def predict(self, X) -> np.ndarray
    def save(self, path) -> None
    def load(self, path) -> None


# 服务接口（src/services/forecast_service.py）
class ForecastService:
    def train(self, selected_models, epochs, batch_size) -> Dict
    def predict_by_horizon(self, horizon) -> Dict


# 业务逻辑接口（src/logic/__init__.py）
class BusinessLogic:
    @staticmethod
    def run_full_pipeline(...) -> Dict
    @staticmethod
    def train_model(...) -> Dict
    @staticmethod
    def predict(...) -> Dict
    @staticmethod
    def get_trade_advice(...) -> Dict
```

## 八、常见问题解答

### Q1: 程序从哪里开始执行？

**A:** 取决于启动方式：
- `python main.py` → 从 `main()` 函数开始
- `streamlit run app.py` → 从 `app.py` 顶部开始，执行到 `main()`

### Q2: src 目录的嵌套关系是什么？

**A:** 依赖关系（从上层到下层）：
```
app.py (前端)
  ↓
logic/ (业务逻辑)
  ↓
services/ (服务编排)
  ↓
models/ + data_layer/ (模型 + 数据)
  ↓
core/ (核心工具)
```

### Q3: 如何追踪代码执行？

**A:** 查看日志文件 `logs/app.log`，包含完整的执行记录。

### Q4: 模型文件在哪里？

**A:** `models/` 目录下：
- `lstm_model.bin` - LSTM 模型
- `gru_model.bin` - GRU 模型
- `xgboost_model.bin` - XGBoost 模型
- `moirai_model.bin` - Moirai 模型
- `stacking_meta.bin` - Stacking 元学习器
