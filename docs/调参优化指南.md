## 二、模型超参数调优

### 2.1 LSTM 模型调参

**文件位置：** `src/models/lstm_model.py`

```python
class LSTMForecastModel(BaseForecastModel):
    def __init__(self, input_dim: int, hidden_dim: int = 64, num_layers: int = 2):
        #              可调参数 ↑              ↑
```

**可调参数列表：**

| 参数 | 默认值 | 推荐范围 | 说明 |
|------|--------|----------|------|
| `hidden_dim` | 64 | 32, 64, 128, 256 | 隐藏层维度，越大模型越复杂 |
| `num_layers` | 2 | 1, 2, 3 | LSTM 层数，越深捕捉能力越强 |

**训练时可传参数：**

| 参数 | 默认值 | 推荐范围 | 说明 |
|------|--------|----------|------|
| `epochs` | 20 | 10-100 | 训练轮数 |
| `batch_size` | 64 | 16, 32, 64, 128 | 批次大小 |
| `lr` | 1e-3 | 1e-4 ~ 1e-2 | 学习率 |

**调参示例：**

```python
# 方法 1：直接修改默认值（推荐）
# src/models/lstm_model.py
class LSTMForecastModel(BaseForecastModel):
    def __init__(self, input_dim: int, hidden_dim: int = 128, num_layers: int = 3):
        # 修改默认值
        ...

# 方法 2：训练时传入参数
# main.py 或 app.py
service.train(
    selected_models=["lstm"],
    epochs=50,           # 增加训练轮数
    batch_size=32,       # 减小批次大小
)

# 方法 3：修改 model_registry.py 中的实例化参数
# src/models/model_registry.py
def build_model_registry(input_dim: int, runtime: RuntimeConfig):
    return {
        "lstm": LSTMForecastModel(
            input_dim=input_dim,
            hidden_dim=128,    # 修改这里
            num_layers=3       # 修改这里
        ),
        ...
    }
```

**调参建议：**

1. **先调学习率**：从 1e-3 开始，如果 loss 不下降尝试 1e-4
2. **再调隐藏层维度**：如果欠拟合增加 hidden_dim，过拟合减小
3. **最后调层数**：一般 2 层足够，3 层可能过拟合

### 2.3 XGBoost 模型调参

**文件位置：** `src/models/xgboost_model.py`

```python
class XGBoostForecastModel(BaseForecastModel):
    def __init__(self):
        self.model = XGBRegressor(
            n_estimators=300,      # 可调
            learning_rate=0.05,    # 可调
            max_depth=6,           # 可调
            subsample=0.9,         # 可调
            colsample_bytree=0.9,  # 可调
            objective="reg:squarederror",
            random_state=42,
            n_jobs=1,
        )
```

**可调参数列表：**

| 参数 | 默认值 | 推荐范围 | 说明 |
|------|--------|----------|------|
| `n_estimators` | 300 | 100-500 | 树的数量，越多越好但慢 |
| `learning_rate` | 0.05 | 0.01-0.2 | 学习率，越小需要更多树 |
| `max_depth` | 6 | 3-10 | 树的最大深度 |
| `subsample` | 0.9 | 0.7-1.0 | 样本采样率，防止过拟合 |
| `colsample_bytree` | 0.9 | 0.7-1.0 | 特征采样率 |

**调参建议：**

1. **先调树的数量和学习率**：n_estimators=300, learning_rate=0.05 是不错的起点
2. **再调 max_depth**：如果过拟合减小 depth，欠拟合增加
3. **最后调采样率**：如果过拟合降低 subsample 和 colsample_bytree

## 四、融合权重调优

**文件位置：** `src/services/forecaster_manager.py`

```python
HORIZON_CONFIGS: Dict[str, HorizonConfig] = {
    "day": HorizonConfig(
        ...
        weights={
            "lstm": 0.40,    # 调整这些权重
            "gru": 0.35,
            "xgboost": 0.15,
            "moirai": 0.10,
        },
    ),
    ...
}
```

**调参逻辑：**

1. **短期预测（day）**：如果 LSTM 表现好，增加 lstm 权重
2. **中期预测（week）**：如果 XGBoost 表现好，增加 xgboost 权重
3. **长期预测（month）**：如果 Moirai 表现好，增加 moirai 权重

**权重调整规则：**
- 权重和必须为 1
- 根据验证集 MAE/RMSE 动态调整
- 表现好的模型权重增加 0.05-0.10

### 6.2 使用 Optuna 自动调参

```python
# 安装：pip install optuna

import optuna

def objective(trial):
    # 定义参数搜索空间
    hidden_dim = trial.suggest_categorical("hidden_dim", [32, 64, 128, 256])
    num_layers = trial.suggest_int("num_layers", 1, 4)
    lr = trial.suggest_float("lr", 1e-4, 1e-2, log=True)
    
    # 训练模型
    model = LSTMForecastModel(
        input_dim=input_dim,
        hidden_dim=hidden_dim,
        num_layers=num_layers
    )
    model.train(X_train, y_train, X_val, y_val, epochs=20, lr=lr)
    
    # 返回评估指标
    val_pred = model.predict(X_val)
    mae = np.mean(np.abs(y_val - val_pred))
    
    return mae

# 创建研究并优化
study = optuna.create_study(direction="minimize")
study.optimize(objective, n_trials=50)

print(f"最优参数：{study.best_params}")
print(f"最优 MAE: {study.best_value}")
```

## 八、常见问题解答

### Q1: 调参后需要重新训练所有模型吗？

**A:** 是的，修改模型超参数后需要重新训练。但可以使用以下策略加速：
1. 先用小数据量（如 1000 条）快速验证参数效果
2. 确认参数有效后再用全量数据训练

### Q2: 如何判断是过拟合还是欠拟合？

**A:** 看训练集和验证集的 loss 差异：

| 情况 | train_loss | val_loss | 解决方案 |
|------|------------|----------|----------|
| 欠拟合 | 高 | 高 | 增加模型复杂度（hidden_dim, num_layers） |
| 过拟合 | 低 | 高 | 减小模型复杂度，增加正则化 |
| 正常 | 低 | 低 | 参数合适 |

### Q3: 融合权重需要重新训练吗？

**A:** 不需要。融合权重只是改变各模型的输出比例，不影响模型本身。修改后直接运行预测即可看到效果。

### Q4: 调参的最佳顺序是什么？

**A:** 推荐顺序：
1. **学习率**（影响最大）
2. **模型容量**（hidden_dim, num_layers）
3. **训练参数**（epochs, batch_size）
4. **融合权重**
